Column schema
-------------
  field       reference field name (e.g. REF_1324-28)
  footprint   beam footprint name (from schedblock metadata, e.g. closepack36)
  variant     calibration variant: 'bpcal' (bandpass cal) or 'lcal' (leakage cal)
              note: footprint is constant across all rows in a single-manifest run;
              rows from different footprints (different manifests) can be safely appended
  beam        ASKAP beam index, 0-based (0-35 for closepack36)
  mean_dQ     mean fractional Stokes-Q leakage dQ/I (%), signed, averaged
              across all SB_REF x ODC observations for this field/variant/beam
  std_dQ      standard deviation of dQ/I (%) across observations
  mean_dU   mean fractional Stokes-U leakage dU/I (%), signed
  std_dU    standard deviation of dU/I (%) across observations
  n_obs       number of (SB_REF, ODC) data rows averaged for this beam

Physical meaning
----------------
  dQ = (Q_measured / I_measured) x 100  -- on-axis Stokes-Q leakage in percent
  dU = (U_measured / I_measured) x 100  -- on-axis Stokes-U leakage in percent
  For an unpolarised calibrator (1934-638), any measured Q or U is pure leakage.
  A positive mean_dQ means the X-feed is systematically stronger than the Y-feed
  for that beam (see gain_calibration_strategy.html for the full derivation).
